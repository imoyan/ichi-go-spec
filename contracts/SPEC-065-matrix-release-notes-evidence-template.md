# SPEC-065: Matrix Release Notes Evidence Template

Status: draft
Feature profile: core
Canonical: yes

## Purpose

Define the release notes evidence template required for any Houra release that
mentions Matrix v1.18 support.

## Scope

This contract is Matrix-defined, not Houra-defined. It defines required release
notes fields for supported Matrix spec version, supported domains, excluded
domains, room versions, excluded unstable MSCs, and implementation evidence. It
does not generate release notes by itself.

## Matrix reference

- Matrix specification version: `v1.18`
- Source: <https://spec.matrix.org/v1.18/>
- Release note: <https://matrix.org/blog/2026/03/26/matrix-v1.18-release/>
- Checked at: 2026-05-10T23:24:00+09:00
- Timezone: Asia/Tokyo

## Required sections

Any release notes that mention Matrix support must include:

- `Matrix spec version`
- `Supported Matrix domains`
- `Excluded Matrix domains`
- `Supported room versions`
- `Excluded unstable MSCs`
- `Implementation evidence`
- `Known gaps`
- `Advertisement decision`

`Implementation evidence` links must point to pass/fail artifacts generated by
`SPEC-062`, `SPEC-063`, and `SPEC-064` adoption gates. Human prose may summarize
the evidence, but it must not replace artifact refs.

## Evidence link format

Each evidence link record contains:

- `repo`
- `ref`
- `domain`
- `gate`
- `status`
- `artifact`
- `issue`

Supported domains require `status=pass`. Excluded domains may use `not-run`,
`fail`, or `not-applicable`, but must also include a known-gap issue or explicit
out-of-scope reason.

## Compatibility boundaries

- This contract defines release notes content only.
- This contract must not by itself widen `GET /_matrix/client/versions`
  advertisement beyond `SPEC-064`.
- After this spec PR is merged, create adoption issues for `houra-server` and
  `houra-client`. Create an `houra-labs` issue only if shared-core release
  artifacts begin publishing Matrix support claims.
